What this is

Build an “Ask-Me” chat — a Claude Artifact grounded in your own real material (CV, projects, services, talks) that answers questions live. No code editor, no server, no API key: Claude builds it inside the chat, and it runs on the viewer’s own Claude login. Pick one of the five tasks below, ground it in your real information, publish it, and prove it by asking it three real questions.

Pick a problem statement

Each one below is a ready-to-run task — a real scenario, clear steps, and the questions it has to answer. These five are just starting points: feel free to use any other problem statement of your own. To see the finished shape, open its sample using the link on each card. Note: a sample shows the interface and its material, but it only answers live once it is opened as an artifact inside claude.ai — that is where Claude is reachable.

How it works

Pick a task and open its sample from the link on its card to see the shape of the finished thing. Or bring your own idea.

Gather your own real material as plain text — that is the only thing your assistant is allowed to say.

In Claude, turn on “AI-powered artifacts” (Settings → Capabilities), then ask Claude to build the Ask-Me chat grounded only in your material.

Publish the artifact and test three real questions — each must answer correctly from your own material. Anyone opening the link needs their own Claude account for it to answer live.

Push your artifact code, the published share link and a short README (what it is · who can use it live) to ho2-askme-assistant — that is your deliverable.

Build it in fewer tokens

Your Claude plan has a usage limit that resets every few hours, and the fastest way to burn it is twenty small vague messages. One complete prompt beats twenty small ones — it tells Claude the persona, the rules, the layout and the grounding in a single message, so it has nothing to ask you back. Copy the prompt below, fill in the parts in <angle brackets>, and paste it into a new chat.

The one-shot build prompt
Works for any of the five problem statements — and for your own. Fill in the four bracketed parts and send it once.
Build a single-file HTML "Ask-Me" Claude Artifact.

WHO IT SPEAKS FOR: <one line — e.g. the campaign and offer desk for Rally Sportswear>
WHO ASKS IT: <one line — e.g. our sales, support and social teams>

Behaviour rules:
- Answer ONLY from the FACTS below. Never invent anything that is not in them.
- Warm, specific and concise — 2 to 4 sentences.
- Quote names, numbers, dates and terms EXACTLY as written in the FACTS.
- If the answer is not in the FACTS, say "<your fallback line>" instead of guessing.

Layout: a clean page with a title, a short About, the FACTS shown as readable
sections a human can scan, and a chat box with 3 suggested starter questions.

Answer live with window.claude.complete(prompt) — no API key. If that is not
available, show a friendly note telling the visitor to open this inside claude.ai
and sign in.

FACTS:
<paste your material here — this is the ONLY thing it is allowed to say>
Why this is cheap: Claude gets the persona, the rules, the layout and the grounding in one message. There is no back-and-forth to pay for, and the first artifact it produces is usually the one you keep.
When it answers something wrong
It got <question> wrong. Add this to the FACTS:
<the missing fact>
Change nothing else.
A wrong answer is almost always a gap in the FACTS, not a bad model. Patch the grounding, not the prompt.
When it guesses
It guessed when it did not know. Make it refuse:
if the answer is not in the FACTS, reply exactly
"<your fallback line>" and nothing else.
Refusing well is the whole point of the exercise. An assistant that invents a price is worse than no assistant.
When you just want it to look different
Change the accent colour to <hex> and the starter
questions to:
1. <q>  2. <q>  3. <q>
Leave everything else exactly as it is.
Tweak, never regenerate. "Change only X" costs a fraction of "build it again, but…".
Four habits that make your plan last
  • One artifact, one chat. Every turn in a long chat re-reads everything above it — a chat you have been in all afternoon is the most expensive place to ask a small question.
  • Paste your FACTS once. After that just say “the FACTS” — Claude can already see them.
  • Tweak, don’t regenerate. “Change only the accent colour” is cheap. “Build it again but nicer” pays for the whole page a second time.
  • When it works, publish it. Get the link, then go experiment in a fresh chat — don’t risk the version that already works.
Ship it to GitHub

Your deliverable is a repo plus a published artifact link. Keep the repo small and honest — four files is all this needs.

ho2-askme-assistant/
├─ index.html      ← the artifact code, exactly as Claude built it
├─ grounding.md    ← the FACTS you gave it — the material it is allowed to say
├─ README.md       ← what it is · your published link · who can use it live
└─ .gitignore

In claude.ai, open your artifact’s “···” menu and copy or download its code. Save it as index.html.

Save the exact material you grounded it in as grounding.md. This is what makes your work checkable — the assistant may only say what is in this file.

Write the README from the template below and paste in your published artifact link.

Create an empty repo called ho2-askme-assistant on GitHub, then run the commands below. git branch -M main matters — older Git still calls the first branch master, and the push would fail.

Push it
git init
git add .
git commit -m "HO2: Ask-Me assistant"
git branch -M main
git remote add origin https://github.com/<your-username>/ho2-askme-assistant.git
git push -u origin main
README template
# Ask-Me Assistant — <what it answers questions about>

**Published artifact:** <paste your claude.ai share link here>

## What it is
A Claude Artifact that answers questions about <subject>, grounded only in
`grounding.md`. It answers live by calling Claude at runtime — no API key,
no server.

## Who can use it live
Anyone with a Claude account. When someone asks it a question they sign in to
their **own** Claude account and the usage counts against theirs. It is not an
anonymous public bot.

## What it will not do
It never invents <prices / dates / claims>. If the answer is not in the
grounding, it says so instead of guessing.
One thing that surprises everybody
  • You can switch on GitHub Pages (repo Settings → Pages → Deploy from branch → main) and your page will load — but the chat will not answer there. window.claude.complete() only exists inside claude.ai. GitHub Pages shows the interface; the published artifact link is the thing that actually answers. Put that link at the top of your README.
